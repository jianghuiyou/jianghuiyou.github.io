设计模式系列（二）——单例模式

单例模式应该算是所有设计模式里类图最简单、实现起来代码最短的模式，所以也是各大互联网公司面试的热点。比如，实现一个线程安全的单例模式，单例模式和全局变量的区别。其实，写出一个漂亮的单例模式并非很容易，希望看完这篇文章的人都能立马自己去实现一个漂亮的单例模式。

单例模式，顾名思义也就是一个类只能有一个实例化对象。这在资源类型中常常被应用，为了减少不必要的性能损坏以及避免多个对象的不同步问题，比如线程池、数据库连接池、windows中的垃圾回收站、任务管理器、注册表等等。

  2. 单例模式的灵活性，它可以extends，也可以实现一个interface，这样可以作为正常的object参数传递到其他函数中。

读《毛泽东传》有感

前不久马云的一句话引起了轩然大波，“我这辈子最大的错误就是创建了阿里巴巴”，大多人都觉着这逼装得飞起啊。其实，为什么社会永远是金字塔顶尖的人极少极少呢，一个是很难往上爬，再者便是维持自己的地位和权利真的太费心啊。毛主席真正成为国家一把手之后，也不会像是秦始皇一样说一不二，一心想要的大跃进遭到彭德怀国防部长的强烈反对，搞掉彭之后，又和苏联老大哥关系闹崩，遭到国家高层一众人的反对。总会想想刘少奇这位二把手地位不断提升，会不会盖过自己？林彪对自己过度奉承是否别有用心？国家整体是要亲近苏联还是亲近美国？国家的接班人到底选谁最合适？自己去世之后国家会不会暴乱？这些问题都需要深思熟虑，而且每一个决策都会影响整个国家，也能让自己从国家元首到跌落深渊甚至命丧黄泉。其实想想，马云身在商界顶峰也一定有自己每天的烦恼，而又无法抽身、常人难以理解的苦衷。

设计模式系列（一）—— 观察者模式

之前为了更好地理解Java IO好好学习了装饰者模式，现在在看Junit4的时候发现太多的设计模式都需要了解，于是打算开始好好地学习下各种常用的设计模式。

只掌握继承、接口、多态这些基础的面向对象知识已经不足以设计出优雅、健壮的代码。其实设计模式，说白了就是面向对象的工程师前辈们总结出来的代码组织方式，所有的设计模式都主要为了达到以下三条目的或在三者之间更好地权衡：

  1. 可复用性（相同结构的代码最大化地共用）
  2. 可扩展性（便于增加新需求）
  3. 易维护性（修改某一个模块尽量不牵动或者少牵动其他模块，也成为松耦合）

观察者模式在JDK源码中大量地应用，尤其是在图形界面方面，实现了数据模型和显示界面的松耦合，也就是MVC模式中的Model和View。观察者模式其实很好理解，我们可以类比生活中订阅报纸的例子，当我们对报纸内容感兴趣的时候，向报社订阅报纸服务，当报纸更新的时候报社便会及时通知到所有订阅者。

观察者模式定义了对象之间一对多的依赖，当一个对象状态改变时，它的所有依赖者便会得到通知并更新。

观察者模式的类图如下：

Subject类比抽象出来的报社，没有具体的实例，而其子类ConcreteSubject就好比人民日报，当然也可以有其他报社。Observer仿佛是对阅读报纸感兴趣这个特性的接口，ConcreteSubject就是一类实现了该接口的人群，例如对报纸感兴趣的学生类。这样有学生订阅报纸并有新的报纸更新的时候，报社会通知所有订阅者。

而能进行主动通知的原因在于抽象类Subject中有所有订阅者对象的引用，在状态发生改变的时候自然可以通过该引用调用Observer类的update函数。下面是我实现的一个最基础的观察者模式代码，强烈建议自己实现一个，回头再理解。

Java IO设计原理——装饰者模式

第一次看Java IO相关讲解的时候，几乎每个人都会被那么多各种各样的输入输出流的类搞蒙，做一个IO传输需要创建很多个类的实例，让人难以接受。其实JAVA先驱者应用的就是装饰者模式这种设计模式。此文就是为大家解答JAVA IO这么设计背后的原理，并非讲解IO类的使用方法。

装饰者模式的功能和继承类似，都是为了扩展类的功能。继承通过子类继承超类的功能，并添加或覆盖自己特有的功能，这也符合面向对象的思想。但是，简单粗暴地继承会带来最直观的问题就是类爆炸，产生太多太多的类。而装饰者模式，是将需要扩展的这些功能封装成多个装饰类，这些每一个功能在代码运行时候可以灵活地组合，而不是通过继承超类在编译时候决定类的功能。

函数指针在本质上与其他基本类型的指针其实是一样的，只不过长得更加复杂，让初学者望而生畏，下面我来分析讲解下函数指针和回调函数。如果你能耐心看完，相信对你理解它们一定大有帮助。

这里声明一个指向整形数据的指针，pa里存放的是整形变量a的地址，可以通过指针pa间接调用a，与此类似。

上面先声明了一个函数hello，然后又声明了一个函数指针，指向的函数具有“返回值是整形变量，形参是字符指针”的特点。最后将hello这段函数二进制代码的首地址（可以直接用hello表示）赋给phello函数指针，这样就可以通过phello来间接调用hello函数。
